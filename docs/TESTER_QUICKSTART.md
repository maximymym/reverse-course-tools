# Tester Quickstart — правим Lua без пересборки

Перезагрузка Lua скриптов идёт **без пересборки DLL и без рестарта Dota**. Правишь файл → нажимаешь кнопку → через 500 мс боты уже действуют по новым правилам.

---

## Где лежат скрипты

```
C:\temp\andromeda\scripts\bots\
├── bot_controller.lua              ← ГЛАВНЫЙ — 11-приоритетная логика, anti-ban, config
├── bot_generic.lua                 ← Valve-style fallback (если bot_controller вернул false)
├── ability_item_usage_generic.lua  ← дефолтный кастинг заклинаний
├── lane_config.lua                 ← распределение героев по линиям
└── BotLib\hero_<name>.lua          ← специфичная логика per-hero (опционально)
```

Можно открыть прямо в Блокноте, VS Code, Notepad++ — любом текстовом редакторе с подсветкой Lua.

## Как применить изменения (3 способа)

### 1. Кнопка "Reload Lua" в DotaFarm (рекомендуется для 5 ботов)

В главном окне `DotaFarm.exe` в топбаре есть голубая кнопка **"Reload Lua"** рядом с **Pause All**.
Один клик — все запущенные `dota2.exe` перезагружают скрипты одновременно.

В логе orchestrator появится: `Reload Lua: flag sent to N bot(s)`.

### 2. F7 в окне Dota (один инстанс)

Фокус на активное окно Dota 2 → нажать **F7** → в этом инстансе Lua перезагрузится.
Удобно если отлаживаешь один бот.

Legacy: **F9** всё ещё работает (для обратной совместимости со старыми скриптами/записями).

### 3. `StartFarm` заново

Выключить ботов (**Stop Farm**) и включить снова. Это полный рестарт DLL и Lua, обычно не нужен — используется если что-то совсем сломано.

## Что смотреть в логах

### `C:\temp\andromeda\botbrain.log`
Главный лог DLL+Lua. Смотри последние строки когда боты ведут себя странно.

Важные маркеры:
- `=== BotBrain Init ===` — старт DLL
- `=== HOT RELOAD ===` — видно при нажатии Reload Lua / F7 / F9
- `[CONTROLLER] tick=N action=...` — что делает Lua каждый тик
- `[LUA ERROR]` — синтаксическая/runtime ошибка в .lua. **Не крашит бота** — DLL падает в C++ fallback.

### `DotaFarm.log` (рядом с `DotaFarm.exe`)
Лог orchestrator. Здесь видно:
- Запуск/остановка ботов
- Отправка `Reload Lua` флагов
- GSI + GC события, pause/resume

## Первая правка — "hello world"

1. Открой `C:\temp\andromeda\scripts\bots\bot_controller.lua` в редакторе.
2. Найди строку:
   ```lua
   critical_hp     = 0.15,   -- <15% → фонтан
   ```
3. Поменяй на:
   ```lua
   critical_hp     = 0.50,   -- <50% → фонтан
   ```
4. Сохрани файл.
5. В DotaFarm GUI нажми **Reload Lua**.
6. В `botbrain.log` через секунду появится `=== HOT RELOAD ===`.
7. В игре боты теперь начнут убегать в фонтан при **HP < 50%** (вместо 15%).
8. Верни `0.15` обратно и снова **Reload Lua**.

Если на шаге 6 строка не появилась — открыт неправильный инстанс Dota (Reload Lua шлёт флаг только живым ботам).

## Если сломал скрипт

1. **Dota не крашится** — в DLL есть C++ fallback (`ThinkCppFallback`). Бот продолжит вести себя по дефолту Valve, просто тупее.
2. В `botbrain.log` увидишь `[LUA ERROR] <описание ошибки>`.
3. Исправь .lua → **Reload Lua** → ошибка должна уйти.
4. Если совсем запутался — скачай свежий `DotaFarm.zip` с сервера, там оригинальные скрипты.

**Никогда не крашит Dota**, если только синтаксис не сломал сам Lua VM на этапе парсинга (тогда весь state зависнет в OFF, но dota останется рабочей).
